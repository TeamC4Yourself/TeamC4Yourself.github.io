Preface

Team C4Yourself will make an application for Festispec, which inspectors from Festispec can use to organize inspections for festivals. Users will be able to add,view,delete and update inspections on site on a smartphone, because the application will store the data and push it online as soon as the user is connected to the internet again. Managers can also use this application to view how well their employees are doing and to request reports. Customers of Festispec will be able to add new festivals and view all inspections from those festivals.

Wireframes

To give an example what the application will look like, here are three wireframes with some of the most important functionality.

Dashboard

The dashboard serves the purpose of allowing the user to access the three core elements of the application. Namely, the customers and their profiles. The festivals and their inspections, as well as the profiles of any inspectors.

Pictures:

Upper left corner: A photo of the inspector.

Pictures under my inspections: logo of the festival.

Pictures under customers: a logo/picture of the company.

Pictures under festivals: a logo of the festival.

Pictures under inspectors: pictures of the inspectors.

The image in the upper right corner is actually a mail icon.

View Inspection Profile

An inspection profile shows the details about an actual inspection. Any details such as what needs to be done are listed here, as well as notes for any part of an inspection. This is naturally accompanied by any other relevant information for an inspection.

Pictures:

Upper left corner: A photo of the inspector.

Pictures under my inspections: logo of the festival.

The image in the upper right corner is actually a mail icon.

Picture next to the inspection details: festival logo

View inspector profile

The inspector profile lists all relevant information which includes, but is not limited to, their personal details, which inspections they are currently assigned to and the status of these inspections.

Pictures:

Upper left corner: a photo of the inspector.

Pictures under my inspections: logo of the festival.

Festival overview

The festival overview gives a clear view of all the inspections and their statuses, a map of the festival in the middle and additional info. The additional info are things like closing times of the festivals, the active days etc.

Research

To get a clear view of what technologies already exist that are required for this project C4Yourself has conducted a small scaled research to see whether there are already existing applications or frameworks that are available to us that contain functionalities that fully or partially fulfill a requirement as stated above. This research is split between applications and packages/frameworks.

Pre-existing applications

To be included in this long list an application must match to one or more of the following criteria:

  • The application adds on to the structure of the code.
  • The application has a functionality that is fully or partially fulfilling a requirement.
  • The application makes a recurring job in the development phase easier.

Long List

  1. E-boekhouden.nl - Starters
  2. Exact Online Boekhouden Geavanceerd
  3. Rompslomp PRO
  4. Microsoft Excel
  5. Snelstart
  6. King Business Software
  7. Afas
  8. Twinfield
  9. Reeleezee
  10. Davilex

Short List

  1. Microsoft Excel

Excel allows for the efficient storage of data in the form of tables and graphs, which are required within our software solution. The fact that this is already in use by Festispec makes this an even easier choice.

  1. King Business Software

King Business Software has software responsible from the administration of a project, which in this case would then be applied to an inspection rather than a project.

  1. Afas

Afas has, just like Excel, the ability to generate graphs based on administrative data, which would be an excellent backup program to Excel itself.

To be included in this long list a package or framework must match to one or more of the following criteria:

  • The package or framework adds on to the structure of the code.
  • The package or framework has a functionality that is fully or partially fulfilling a requirement.
  • The package or framework makes a recurring job in the development phase easier.

Out of this Long list we have made a short list of packages that seem particularly interesting.

Short List

  1. EntityFramework

Entity framework has been chosen because a good connection between database and classes are required and the entity framework provides just that.

  1. NUnit

Furthermore we have decided to pick NUnit for our shortlist as NUnit creates the ability to write tests and easily execute these. NUnit also makes it possible to visually see what tests have passed or failed.  This allows us to provide a compatibility test and easily test this repeatedly.

  1. IdentityManager

Finally we have decided to go with IdentityManager as it supplies an easy to maintain CRUD system for user management. This will make user management in the deployment phase significantly easier and more streamlined.
